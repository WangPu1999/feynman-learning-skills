## What To Learn From It

The useful design pattern is to treat research training as staged capability
building rather than as a list of papers:

1. Build broad foundations first.
2. Follow an existing project to learn depth: paper details, code details,
   experiment details, and weekly discussion.
3. Move into an independent project loop: find a problem, form an idea, run
   experiments, write, present, and revise.
4. Eventually connect projects into a longer research roadmap around important
   questions.

The skill should adapt this pattern to the learner's actual domain rather than
assuming a fixed 3D vision route.

## Transfer Into This Skill

When a learner asks how to improve research ability, ask:

```text
你现在更像在哪个阶段？
1. 打基础：还在补课程、工具和概念地图。
2. 跟项目：正在读论文、跑代码、做实验，学习项目怎么做出来。
3. 做一作项目：需要自己选题、想 idea、做实验、写论文和展示。
4. 建 roadmap：需要把多个项目连接到长期重要问题。
```

Then choose one ability:

- field overview
- paper reading
- idea generation
- experiment design
- failure analysis
- experiment logging
- paper writing
- presentation
- advisor/peer meeting
- long-term roadmap
