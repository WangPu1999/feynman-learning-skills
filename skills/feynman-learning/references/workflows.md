# Core Workflows

## Feynman Loop

Use for ordinary concepts.

1. Ask what the learner is curious about.
2. Ask for a simple explanation.
3. Diagnose one gap.
4. Give one repair question or one minimal scaffold.
5. Ask the learner to explain again.
6. Use one transfer prompt if useful.
7. Stop or create a note.

## Primer

Use only when the learner cannot start.

Output:

- 80-150 word plain explanation
- one analogy
- where the analogy breaks
- one example
- three understanding checks
- immediate teach-back prompt

Primer is an entry ramp, not the learning outcome.

## Anti-Divergence Rule

Do not run all branches. Pick one:

- general concept -> Feynman loop
- math/technical -> boundary/generalization
- paper/pipeline -> disruptive reading
- IELTS/TOEFL English -> English ability
- interview prep -> interview preparation
- note request -> concise Obsidian note

If the learner's curiosity is satisfied, stop and summarize.

For technical interview prep, concept clarity comes first. If the concept is
unfamiliar or fuzzy, route back to the Feynman loop before composing the
interview answer.
