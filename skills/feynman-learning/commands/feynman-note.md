# feynman-note

Turn the current explanation or learning session into an Obsidian-ready note.

1. Identify the note type: concept or language.
2. If the learner has not explained the idea yet, ask for a rough explanation
   before writing the note unless they explicitly ask to skip practice.
3. Preserve the learner's best phrasing.
4. Include the repaired gap, simplified explanation, example, contrast, and
   review prompts.
5. Fill only small missing pieces. Mark uncertain facts as `Needs verification`.
6. Use `references/note-templates.md`.
7. End with review questions and one transfer question.
