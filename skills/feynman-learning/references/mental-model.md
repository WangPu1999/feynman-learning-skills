## Core Model

The video frames the Feynman technique as a learning mindset rather than a note
format. The core claim is that many learners are blocked less by intelligence
than by method. Memorization alone creates familiarity, while explanation
reveals whether the learner can actually use the idea.

The skill should therefore optimize for learner production. A good session makes
the learner speak or write clearer explanations over time. A bad session gives a
beautiful explanation while the learner stays passive.

## Default Loop

Use this loop for concept learning, language learning, and note generation:

```text
Choose one target
-> simplify it in the learner's own words
-> teach it aloud or in writing
-> identify one gap
-> return to source or give a minimal scaffold
-> simplify again
-> repeat creatively in a new form
-> turn the repaired understanding into a note
```

The loop is more important than any single output template.

## Operational Steps

### 1. Start From Simplification

Pick exactly one concept. Ask the learner to explain it with:

- simple words
- short sentences
- everyday examples
- a child-friendly level of clarity

This step strips away prestige vocabulary and reveals hidden gaps. If the
learner cannot simplify, treat that as signal rather than failure.

Agent behavior:

- Ask for one target, not a whole subject.
- Ask for short sentences.
- Ask for an everyday example.
- Flag complex language that hides uncertainty.

### 2. Learn by Teaching

Teaching does not require a real classroom. The learner can explain to:

- a friend
- a camera
- a mirror
- a blank page
- the agent in the chat

The important move is turning private thought into spoken or written language.
When an idea becomes language, weak links become visible.

Agent behavior:

- Ask the learner to explain before giving a full answer.
- If the learner asks for a note too early, ask for a rough explanation first.
- Accept roughness; the point is to expose gaps.

### 3. Return to the Source

After a failed or incomplete explanation, send the learner back to source
material or a minimal scaffold, then ask them to simplify again.

Loop:

```text
explain -> find gap -> review source -> simplify again -> explain again
```

Do not let review become passive rereading. Every return to source should end in
a new explanation.

Agent behavior:

- Use source review only to repair the current gap.
- After review, ask the learner to explain the repaired point in simpler words.
- Do not continue to the next gap until the current one is repaired enough.

### 4. Use Creative Repetition

Repeat the same concept in different forms:

- one everyday analogy
- one diagram or spatial description
- one story
- one contrast pair
- one practical example
- one different context

Variation builds more retrieval paths than rote repetition. For language
learning, ask the learner to use the same expression in multiple situations.

Agent behavior:

- Ask for one alternate representation at a time.
- Use analogy, example, contrast, story, diagram-like description, or a new
  context.
- For language, vary speaker, tone, situation, and collocation.

### 5. Treat Errors as a Map

When the learner gets stuck, contradicts themselves, or gives a broken example,
frame the error as a pointer to the next bridge to build.

Useful response pattern:

```text
This is not a failure. It tells us exactly which bridge is missing: [gap].
Let's repair only that bridge first.
```

Errors are most useful when converted into precise review targets.

Agent behavior:

- Name the error precisely.
- Convert it into one repair target.
- Avoid shaming language.
- Do not let the learner quit just because the first explanation failed.

### 6. Add Effort Intensity

The method works only if the learner actively attempts explanation, revision,
and transfer. Avoid doing all cognitive work for the learner. When the learner
asks for a full answer too early, give a small scaffold and return the task.

Agent behavior:

- Keep scaffolds short.
- End scaffolds with a teach-back request.
- When the learner repeatedly asks to bypass effort, provide a concise answer
  but mark that mastery was not tested.

## Skill Implications

- Ask for learner production early.
- Prefer short prompts over long lectures.
- Use one-gap repair to keep effort focused.
- Turn mistakes into the next learning target.
- End sessions with notes only after explanation improves.
- For language learning, require production in multiple contexts.
